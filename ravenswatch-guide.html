How to Mod Ravenswatch — A Beginner's Guide

Ravenswatch is a roguelite action game by Passtech Games, and like most games it is more fun when you can tweak it. This guide explains, in plain language, how modding the game actually works, how to install and remove mods safely using the Ravenswatch Mod Manager I built, and how to put everything back to normal whenever you want. No prior modding experience needed.

How Ravenswatch mods work

Ravenswatch stores its art, data, and other assets in a "cooked" content folder. Each file's path inside that folder is scrambled with a fixed character cipher, which is why the folder looks like a pile of unreadable names. A mod is simply a replacement for one or more of those files — a new texture, a tweaked stats file, and so on. Because the game happily loads any file that has the correct format, you can change a lot just by swapping files in and out. There is no need to patch the running game in memory for most mods.

Doing this by hand is risky: you have to find the right scrambled filename, keep a copy of the original, and remember exactly what you changed. Forget one step and you can end up reinstalling the whole game. A mod manager exists to handle all of that bookkeeping for you.

The golden rule: always keep a backup

Before changing any game file, make sure you can get the original back. Ravenswatch Mod Manager does this automatically — it copies the original file to a backup before replacing it, and records every change in its own state file. But if you ever mod by hand, copy the file somewhere safe first. This one habit prevents almost every "I broke my game" situation.

Installing a mod with Ravenswatch Mod Manager

The manager removes the manual steps. The typical flow looks like this:

  • Point it at your game. On first run the tool locates your Ravenswatch installation. If it cannot find it, you can set the folder manually.
  • Browse or add a mod. Pick a mod from the registry, or drop a local mod folder into your mods directory.
  • Apply. The manager backs up the affected original files, then copies the mod's files into the right (scrambled) locations.
  • Launch and play. Start the game as usual. Your changes are live.

Because everything is tracked, you can enable several mods and keep a clear picture of what is active.

Removing a mod or restoring the game

This is where a manager really earns its place. To remove a single mod, disable it and the tool restores the original files it replaced. To return the game to a completely clean state, run the restore-all action: every backed-up file is put back and the tracking state is cleared. After that, the game is exactly as it was before you started — you can even verify file integrity through your game launcher and it should report no changes.

Staying safe online

A few sensible habits go a long way:

  • Prefer mods that ship data, not executable scripts. A texture or a stats tweak cannot do much harm; a random executable can.
  • Restore the original game before installing updates, then re-apply your mods afterwards. Game patches can change file formats.
  • Be considerate in multiplayer. Cosmetic and personal tweaks are one thing; changing shared gameplay can affect other players or simply fail to sync.
  • Keep backups. With a manager this is automatic, but it is still the most important rule.

Common questions

Will mods get me banned?

Single-player and cosmetic mods are low risk. As always, respect the game's terms of service, and be cautious about anything that changes online play.

Can I undo everything?

Yes. The whole point of the manager is that every change is backed up and reversible. Restore-all returns the game to its original files.

Do I need to be technical?

No. The desktop app gives you a simple interface. The same features are also available from a command line if you prefer that.

What about game updates?

Restore the game before updating, let the patch install, then re-apply your mods. This avoids conflicts when file formats change.
