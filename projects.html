Ravenswatch Mod Manager

Ravenswatch Mod Manager (RSMM) is a free, open-source tool I built for installing, browsing, and managing community-made mods for Ravenswatch, the roguelite action game by Passtech Games. It runs in the browser and as a native desktop app on Windows, macOS, and Linux. This page is a write-up of what it does and how it works under the hood.

What it does

Modding a game usually means copying files into the right folders by hand, keeping track of what you changed, and undoing it all when something breaks. RSMM removes that friction. It discovers your game install, lets you enable or disable mods from a clean interface, and keeps a record of every change so the original game can be restored with a single action. There are no manual file edits and nothing permanent — disabling a mod puts the original files back exactly as they were.

One-click install

Browse a registry of community mods and apply them without touching the game's folders yourself. Each mod ships as declarative data, not executable scripts.

Safe and reversible

Every original file is backed up before it is replaced, and the manager tracks its own state. Removing a mod — or the whole manager — restores the game completely.

Cross-platform

A single codebase powers the browser version and the desktop apps for Windows, macOS, and Linux, so the experience is the same everywhere.

Open source

The full project — the command-line core, the desktop shell, and the website — is public on GitHub for anyone to read, audit, or contribute to.

How it works

Ravenswatch loads its assets from a cooked content folder where each file's path is obfuscated with a fixed character cipher. RSMM understands that format: it maps a human-readable asset path to its on-disk encoded name, backs up the original, and copies the replacement into place. Because the engine accepts any byte-compatible file, texture and asset swaps work entirely through file replacement — no live memory patching of the game is involved. For Lua-scripted mods on Windows, an optional loader can inject a scripting runtime, but the core texture and data mods need none of that.

A guiding rule of the project is that mods ship data, not code. Instead of every mod bundling its own install script, the shared toolkit handles cooking assets, registering new content, and applying or restoring changes. A new mod is a small declarative manifest plus the assets it provides — which keeps the whole ecosystem easy to review and safe to run.

Tech stack

The install and lifecycle logic is a Python command-line tool with no runtime dependencies. The desktop app is a Tauri 2 shell that bundles that same CLI as a sidecar, so the desktop and the terminal behave identically. The website and registry are built with Next.js, and the documentation and shared packages live in a single monorepo. The native loader for Lua mods is a small C++ component built with CMake.

Why I built it

I wanted modding Ravenswatch to feel approachable for players who are not comfortable editing game files, while keeping it transparent enough that more technical users can see exactly what every change does. Working on it has been a deep dive into reverse-engineering an asset format, designing a reversible install pipeline, and shipping the same core to four different platforms — the kind of end-to-end project I enjoy most.

The project lives at github.com/Ovilli/RavenswatchModManager.
